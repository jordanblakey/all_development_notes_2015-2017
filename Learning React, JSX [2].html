Siblings Passing Props to each other and updating the Parents State (Single Responsibility Principle)

Parent Component
var React = require('react');
var ReactDOM = require('react-dom');
var Child = require('./Child');
var Sibling = require('./Sibling');

var Parent = React.createClass({
  getInitialState: function () {
  return { name: 'Frarthur' };
  },

  changeName: function (newName) {
  this.setState({
  name: newName
  });
  },

  render: function () {
  return (
  <div>
  <Child 
  onChange={this.changeName} />
  <Sibling name={this.state.name} />
  </div>
  );
  }
});

ReactDOM.render(
  <Parent />,
  document.getElementById('app')
);

Child Component
var React = require('react');

var Child = React.createClass({
  handleChange: function (e) {
  var name = e.target.value;
  this.props.onChange(name);
  },

  render: function () {
  return (
  <div>
  <select 
  id="great-names" 
  onChange={this.handleChange}>

  <option value="Frarthur">Frarthur</option>
  <option value="Gromulus">Gromulus</option>
  <option value="Thinkpiece">Thinkpiece</option>
  </select>
  </div>
  );
  }
});

module.exports = Child;

Props vs State purpose
A React component should use props to store information that can be changed, but can only be changed by a different component.
A React component should use state to store information that the component itself can change.

JSX is a syntax extension for JavaScript. It was written to be used with React. JSX code looks a lot like HTML.
What does "syntax extension" mean?
In this case, it means that JSX is not valid JavaScript. Web browsers can't read it!
If a JavaScript file contains JSX code, then that file will have to be compiled. That means that before the file reaches a web browser, a JSX compiler will translate any JSX into regular JavaScript.
Codecademy's servers already have a JSX compiler installed, so you don't have to worry about that for now. Eventually we'll walk through how to set up a JSX compiler on your personal computer.
